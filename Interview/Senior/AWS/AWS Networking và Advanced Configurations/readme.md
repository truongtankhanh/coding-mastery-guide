## AWS Networking:

### Câu Hỏi Phỏng Vấn Cấp Độ Senior về AWS VPC

#### 1. **Virtual Private Cloud (VPC):**

- Mô tả VPC và tại sao nó quan trọng trong môi trường đám mây của AWS?
- Cấu trúc cơ bản của VPC và các thành phần chính như subnets, route tables, và security groups.

#### 2. **Routing và Networking Protocols:**

- Làm thế nào hệ thống routing được thiết lập trong VPC? Giải thích về Route Tables và Network Access Control Lists (NACLs).
- Sử dụng các giao thức mạng như TCP/IP, UDP để quản lý kết nối trong môi trường AWS và trong VPC.

#### 3. **Internet Gateways và NAT Gateways:**

- Sự khác biệt giữa Internet Gateways và NAT Gateways và cách chúng liên kết VPC với Internet.
- Khi nào cần sử dụng Internet Gateways và khi nào cần NAT Gateways trong một VPC?

#### 4. **Elastic IP Addresses và Public/Private Subnets:**

- Elastic IP Addresses có vai trò gì trong AWS và trong VPC?
- Sự khác biệt và mối quan hệ giữa public và private subnets trong cấu trúc VPC.

## Advanced Configurations:

### Câu Hỏi Phỏng Vấn Cấp Độ Senior về AWS Networking và Connectivity

#### 1. **Peering và Direct Connect:**

- **VPC Peering vs. AWS Direct Connect:**
  - VPC Peering là gì? So sánh với AWS Direct Connect.
  - Cách thiết lập kết nối giữa VPC thông qua VPC Peering và kết nối trực tiếp với mạng riêng qua AWS Direct Connect.

#### 2. **Network Access Control và Security Groups:**

- **Quản lý Quyền Truy Cập và Bảo Mật:**
  - Sử dụng Network Access Control và Security Groups để quản lý quyền truy cập mạng và bảo vệ.
  - Quy trình thiết lập và quản lý các rules trong Security Groups và Network Access Control Lists (NACLs).

#### 3. **VPN và Site-to-Site Connectivity:**

- **Loại VPN trên AWS:**
  - Phân biệt giữa các loại VPN trên AWS, ví dụ như IPsec VPN, Client VPN, SSL VPN.
- **Kết nối Site-to-Site VPN:**
  - Hướng dẫn thiết lập kết nối site-to-site VPN để liên kết mạng của bạn với môi trường VPC.

#### 4. **Elastic Load Balancing và Auto Scaling Groups:**

- **Cân Bằng Tải và Tự Động Mở Rộng:**
  - Sử dụng Elastic Load Balancing để cân bằng tải và tăng khả năng sẵn sàng cho ứng dụng.
  - Kết hợp Auto Scaling Groups với Elastic Load Balancing để tự động mở rộng ứng dụng dựa trên tải.
